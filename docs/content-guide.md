# 简历网站内容指南

## 建议披露的信息

| 信息 | 说明 |
|------|------|
| 姓名 | 真名或常用英文名 |
| 职位/职业方向 | 如"前端工程师"、"全栈开发者" |
| 技能栈 | 编程语言、框架、工具及熟练度 |
| 项目经历 | 开源项目、个人项目，概括性描述目标和贡献 |
| 工作经历 | 公司名 + 职位 + 时间段 + 概括性成就 |
| GitHub / LinkedIn | 公开的职业社交链接 |
| 工作邮箱 | 专门用于接收工作联系的邮箱 |
| 所在城市 | 城市级别即可，如"上海" |

## 不建议披露的信息

| 信息 | 原因 |
|------|------|
| 手机号 | 爬虫抓取后会收到骚扰电话 |
| 私人邮箱 | 用专用工作邮箱代替，避免垃圾邮件 |
| 详细住址 | 安全风险，城市级别足够 |
| 身份证号 / 出生日期 | 隐私敏感，无展示必要 |
| 公司内部项目细节 | 架构图、内部系统名、数据量级等可能违反 NDA |
| 薪资期望 | 留给面试环节沟通 |
| 照片 | 除非目标行业看重，否则可能引入偏见 |

## 灰色地带（视情况决定）

| 信息 | 建议 |
|------|------|
| 学历 | 如果是加分项就写，否则用项目和经验说话 |
| 年龄 | 国内简历常见，但公开网站上没必要 |
| 公司名 | 知名公司可以写，小公司可用行业描述代替（如"某头部电商"） |

## 核心原则

公开网站上只放你愿意让陌生人看到的信息。详细的个人信息留给面试环节或私发的 PDF 简历。
